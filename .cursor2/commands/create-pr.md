# Create Pull Request

Creates a request to add your changes to the main project. Think of it like asking to merge your work into the main version. Includes writing a clear description of your changes.

1. **Get ready**
   - Make sure all changes are saved
   - Upload your changes to GitHub
   - Check everything is up to date

2. **Write the description**
   - Look at all the changes made
   - Write a clear summary of what changed
   - Explain why you made these changes
   - List what was added or fixed
   - Mention if anything breaks
   - Explain how you tested it (unit tests, E2E tests with Playwright, manual testing)
   - Link to related issues if any
   - Add pictures if you changed how things look

3. **Create it**
   - Give it a clear title
   - Add helpful tags
   - Set up who should review it

4. **Handle feedback** (if needed later)
   - Read reviewer comments
   - Fix requested changes
   - Respond to questions clearly
   - Update the PR when done

## What to Tell the User

- Explain what a pull request is in simple terms
- Ask them what they changed and why
- Write a clear, helpful description for them
- Show them how to create it on GitHub
- Tell them what happens next
- If there are comments, help them understand and fix them
