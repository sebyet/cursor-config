# Commands Quick Reference

Type `/command-name` and describe what you want in plain English. That's it!

## Commands

**`/setup-project`** - Gets everything ready for a new React Native project. Use when starting fresh.

**`/add-feature`** - Adds something new to your mobile app. Describe what you want, review the plan, then build it.

**`/update-feature`** - Changes something that already exists. Describe what to change, review the plan, then update it.

**`/remove-feature`** - Removes something from your app safely. Describe what to remove, review what will be deleted, then remove it.

**`/customize-app`** - Changes how your app looks (colors, fonts, style). Use when you want to update the design.

**`/start-app`** - Shows your app on your simulator/emulator or device so you can see it. Use when you want to view your app.

**`/test-app`** - Checks if everything works correctly. Use before sharing with others or going live.

**`/fix-issues`** - Fixes problems automatically. Use when something isn't working.

**`/plan-app-flow`** - Creates a plan showing how users will use your app. Use at the beginning of your project.

**`/preview-app`** - Creates a preview build you can share with others to get feedback. Use when you want to show or test your work.

**`/go-live`** - Publishes your app to the App Store and Google Play Store. Use when you're ready to publish.

**`/sync-configuration`** - Updates your Cursor setup from a shared location. Use when updating your setup.

**`/help`** - Suggests the best command for what you want to do. Use when you're not sure which command to use.

## Common Workflows

**Starting a new project:** `/setup-project` → `/customize-app` → `/add-feature`

**Adding something new:** `/add-feature` → `/test-app` → `/preview-app` → `/go-live`

**Fixing problems:** `/fix-issues` → `/test-app`

**Going live:** `/test-app` → `/preview-app` → `/go-live`
