# Analyze Incident

Analyze and document incidents to understand root causes and prevent recurrence.

## What This Command Does

Analyzes incidents by:
- Documenting incident timeline
- Identifying root cause
- Analyzing impact
- Generating post-mortem
- Recommending improvements

## How It Works

1. **Gather Incident Data**
   - Ask about incident details
   - Get timeline of events
   - Understand impact
   - Collect logs and metrics

2. **Analyze Incident**
   - Reconstruct timeline
   - Identify root cause
   - Analyze contributing factors
   - Assess impact

3. **Generate Post-Mortem**
   - Create incident report
   - Document root cause
   - List action items
   - Recommend improvements

## What to Tell the User

- "What incident do you want to analyze?"
- "What happened? Timeline?"
- Analyze incident and generate post-mortem
- "Incident analysis complete! Root cause: [summary]. Action items: [list]"
