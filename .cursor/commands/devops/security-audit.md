# Security Audit

Comprehensive security audit covering code, infrastructure, dependencies, and configurations.

## What This Command Does

Performs security audits by:
- Reviewing code for vulnerabilities
- Checking infrastructure security
- Auditing dependencies
- Reviewing access controls
- Identifying security risks

## How It Works

1. **Define Audit Scope**
   - Ask what to audit (code, infrastructure, etc.)
   - Identify security concerns
   - Understand compliance needs
   - Get access to systems

2. **Perform Security Audit**
   - Review code for vulnerabilities
   - Check infrastructure security
   - Audit dependencies
   - Review access controls
   - Identify risks

3. **Generate Audit Report**
   - Create security report
   - Prioritize vulnerabilities
   - Recommend fixes
   - Provide remediation steps

## What to Tell the User

- "What do you want to audit? (code, infrastructure, dependencies)"
- "What are your security concerns?"
- Perform security audit
- "Security audit complete! Vulnerabilities found: [list]. Priority: [high/medium/low]. Recommendations: [list]"
