# Infrastructure Documentation

Generate and maintain infrastructure documentation for team knowledge and onboarding.

## What This Command Does

Creates infrastructure documentation by:
- Generating infrastructure diagrams
- Documenting infrastructure architecture
- Creating infrastructure runbooks
- Documenting infrastructure decisions
- Maintaining infrastructure wiki

## How It Works

1. **Understand Documentation Needs**
   - Ask about infrastructure to document
   - Identify documentation audience
   - Understand documentation goals
   - Determine documentation format
   - Get context on existing documentation

2. **Generate Infrastructure Documentation**
   - Create infrastructure diagrams
   - Document architecture and components
   - Create network diagrams
   - Document resource relationships
   - Generate documentation structure

3. **Create Runbooks and Guides**
   - Create operational runbooks
   - Document common procedures
   - Create troubleshooting guides
   - Document disaster recovery procedures
   - Create onboarding documentation

4. **Maintain Documentation**
   - Keep documentation up to date
   - Review documentation regularly
   - Update diagrams and guides
   - Document infrastructure changes
   - Maintain documentation wiki

## What to Tell the User

- "What infrastructure needs to be documented?"
- "Who is the documentation audience?"
- "What documentation format do you prefer?"
- Generate infrastructure documentation
- "Infrastructure documentation complete! Diagrams: [created]. Runbooks: [generated]. Wiki: [maintained]"
