# Infrastructure Testing

Test infrastructure configurations and changes to ensure reliability and correctness.

## What This Command Does

Tests infrastructure by:
- Testing infrastructure changes
- Validating configurations
- Testing disaster recovery
- Testing scaling behavior
- Generating test reports

## How It Works

1. **Understand Testing Requirements**
   - Ask about infrastructure to test
   - Identify test scenarios
   - Understand testing goals
   - Determine testing environment
   - Get context on infrastructure changes

2. **Design Test Strategy**
   - Plan test scenarios
   - Design test cases
   - Plan test environment
   - Design test automation
   - Plan test validation

3. **Execute Infrastructure Tests**
   - Test infrastructure changes
   - Validate configurations
   - Test disaster recovery procedures
   - Test scaling behavior
   - Test failover scenarios

4. **Generate Test Reports**
   - Document test results
   - Identify test failures
   - Generate test reports
   - Recommend fixes
   - Document test procedures

## What to Tell the User

- "What infrastructure needs to be tested?"
- "What test scenarios should we run?"
- "What's your testing environment?"
- Execute infrastructure tests
- "Infrastructure testing complete! Tests: [count]. Passed: [count]. Issues: [list]"
