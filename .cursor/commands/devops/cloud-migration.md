# Cloud Migration

Plan and execute cloud migration to move infrastructure and applications to the cloud.

## What This Command Does

Manages cloud migration by:
- Planning cloud migration strategy
- Assessing migration readiness
- Executing migration plan
- Validating migration
- Optimizing post-migration

## How It Works

1. **Assess Migration Readiness**
   - Ask about current infrastructure
   - Identify applications to migrate
   - Assess migration complexity
   - Identify dependencies
   - Determine target cloud provider

2. **Plan Migration Strategy**
   - Choose migration approach (lift-and-shift, re-platform, re-architect)
   - Plan migration phases
   - Identify migration risks
   - Plan rollback strategy
   - Create migration timeline

3. **Execute Migration**
   - Execute migration phases
   - Migrate infrastructure
   - Migrate applications
   - Configure cloud services
   - Document migration steps

4. **Validate and Optimize**
   - Validate migrated infrastructure
   - Test application functionality
   - Optimize cloud resources
   - Monitor post-migration performance
   - Document migration results

## What to Tell the User

- "What infrastructure/applications need to be migrated?"
- "What's your target cloud provider?"
- "What's your migration timeline?"
- Plan and execute cloud migration
- "Cloud migration complete! Phases: [executed]. Validation: [passed]. Optimization: [recommended]"
