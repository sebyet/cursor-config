# Auto Scaling

Configure auto-scaling policies and groups to automatically adjust resources based on demand.

## What This Command Does

Configures auto-scaling by:
- Setting up auto-scaling groups
- Configuring scaling policies
- Setting up scaling triggers
- Configuring scaling metrics
- Testing auto-scaling behavior

## How It Works

1. **Understand Scaling Needs**
   - Ask about scaling requirements
   - Identify resources to scale
   - Understand traffic patterns
   - Determine scaling strategy
   - Get context on cloud provider

2. **Design Auto-Scaling Configuration**
   - Configure auto-scaling groups
   - Set up scaling policies (target tracking, step, simple)
   - Configure scaling triggers
   - Set up scaling metrics
   - Plan scaling limits

3. **Generate Auto-Scaling Configuration**
   - Create auto-scaling group configurations
   - Set up scaling policies
   - Configure metric alarms
   - Set up scaling notifications
   - Document auto-scaling setup

4. **Test and Validate**
   - Test auto-scaling behavior
   - Validate scaling triggers
   - Monitor scaling performance
   - Optimize scaling policies
   - Document scaling behavior

## What to Tell the User

- "What resources need auto-scaling?"
- "What are your scaling requirements?"
- "What cloud provider are you using?"
- Configure auto-scaling policies
- "Auto-scaling setup complete! Groups: [configured]. Policies: [created]. Testing: [recommended]"
