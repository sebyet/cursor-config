# Infrastructure Backup

Backup infrastructure configurations, state, and critical data.

## What This Command Does

Manages infrastructure backups by:
- Backing up infrastructure state
- Backing up configuration files
- Setting up automated backups
- Testing backup restoration
- Documenting backup procedures

## How It Works

1. **Understand Backup Requirements**
   - Ask about infrastructure to backup
   - Identify critical configurations
   - Understand backup frequency needs
   - Determine backup storage location
   - Get context on existing backup strategy

2. **Design Backup Strategy**
   - Plan backup scope and frequency
   - Choose backup storage solution
   - Configure backup retention
   - Plan backup encryption
   - Design backup verification

3. **Set Up Automated Backups**
   - Configure backup automation
   - Set up backup schedules
   - Configure backup storage
   - Set up backup monitoring
   - Document backup configuration

4. **Test and Validate Backups**
   - Test backup restoration
   - Validate backup integrity
   - Document restoration procedures
   - Set up backup alerts
   - Create backup runbook

## What to Tell the User

- "What infrastructure needs to be backed up?"
- "What's your backup frequency requirement?"
- "Where should backups be stored?"
- Set up infrastructure backup strategy
- "Infrastructure backup setup complete! Scope: [defined]. Automation: [configured]. Testing: [recommended]"
