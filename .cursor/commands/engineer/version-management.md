# Version Management

Manage semantic versioning, changelogs, and version tags.

## What This Command Does

Manages versions by:
- Applying semantic versioning (semver)
- Creating version tags
- Updating changelogs
- Managing version branches
- Documenting version history

## How It Works

1. **Understand Version Needs**
   - Ask about version type (major, minor, patch)
   - Identify current version
   - Understand versioning strategy
   - Determine changelog format

2. **Manage Version**
   - Calculate next version
   - Create version tag
   - Update changelog
   - Update version files

3. **Document Changes**
   - Update version history
   - Create release tag
   - Document version changes
   - Notify team

## What to Tell the User

- "What type of release is this? (major, minor, patch)"
- "What changes are in this version?"
- Manage version and update changelog
- "Version management complete! Version: [new]. Tag: [created]. Changelog: [updated]"
