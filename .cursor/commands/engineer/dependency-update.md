# Dependency Update

Update dependencies, apply security patches, and analyze breaking changes.

## What This Command Does

Manages dependencies by:
- Updating dependencies
- Applying security patches
- Analyzing breaking changes
- Testing updates
- Generating update reports
- Managing version conflicts

## How It Works

1. **Understand Update Needs**
   - Ask about dependencies to update
   - Identify security concerns
   - Understand update scope
   - Check for breaking changes

2. **Analyze Dependencies**
   - Check current versions
   - Find available updates
   - Identify security patches
   - Analyze breaking changes
   - Check compatibility
   - Review changelogs

3. **Plan Updates**
   - Create update strategy
   - Prioritize security patches
   - Plan breaking change migration
   - Estimate update effort
   - Create rollback plan

4. **Apply Updates**
   - Update dependencies
   - Apply security patches
   - Handle breaking changes
   - Update code if needed
   - Fix compatibility issues

5. **Verify Updates**
   - Run tests
   - Check functionality
   - Verify security fixes
   - Test breaking changes
   - Generate update report

## What to Tell the User

- "What dependencies do you want to update?"
- "Any security concerns?"
- Analyze dependencies
- "Dependency analysis complete! Updates available: [count]. Security patches: [count]. Breaking changes: [list]"
- Apply updates
- "Dependencies updated! Changes: [list]. Tests: [status]"
