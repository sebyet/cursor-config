# Lint Fix

Fixes code formatting and style issues automatically. Cleans up your code to match project standards and makes it easier to read and maintain.

1. **Find code style issues**
   - Check for formatting problems (indentation, spacing)
   - Check for style violations (prettier, eslint)
   - Check for unused imports or variables
   - Check for inconsistent code style
   - Tell the user what you found

2. **Fix code style issues**
   - Fix formatting automatically
   - Fix style violations
   - Remove unused imports
   - Remove unused variables
   - Fix inconsistent code style
   - Make code consistent with project standards

3. **Verify code is clean**
   - Run linter to confirm all issues fixed
   - Verify formatting is correct
   - Make sure nothing broke
   - Tell them when it's done!

## What to Tell the User

- Explain you're checking code style
- List what issues you found (e.g., "Found formatting issues and 2 unused imports...")
- Fix issues automatically
- Show what you're fixing
- Verify code passes linting after fixes
- Tell them when all style issues are fixed
