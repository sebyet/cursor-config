# Debug Issue

Debug production issues by reproducing bugs, analyzing error logs, and identifying root causes.

## What This Command Does

Helps debug issues by:
- Reproducing bugs
- Analyzing error logs
- Identifying root causes
- Debugging memory issues
- Performance debugging
- Generating debugging reports

## How It Works

1. **Understand the Issue**
   - Ask about the bug or issue
   - Get error messages or logs
   - Understand when it occurs
   - Identify affected users
   - Get reproduction steps

2. **Reproduce the Issue**
   - Set up reproduction environment
   - Reproduce the bug
   - Capture error details
   - Log relevant information
   - Identify trigger conditions

3. **Analyze Error Logs**
   - Parse error logs
   - Identify error patterns
   - Find stack traces
   - Analyze error frequency
   - Correlate with user actions

4. **Identify Root Cause**
   - Trace error to source
   - Identify code path
   - Find data issues
   - Check configuration
   - Verify dependencies

5. **Create Fix Plan**
   - Propose fix approach
   - Identify affected code
   - Plan test strategy
   - Estimate fix complexity
   - Create implementation steps

6. **Implement Fix**
   - Apply bug fix
   - Add error handling
   - Add logging if needed
   - Test the fix
   - Verify resolution

## What to Tell the User

- "What issue are you experiencing?"
- "What error messages do you see?"
- "When does this occur?"
- Reproduce and analyze issue
- "Issue analysis complete! Root cause: [details]. Fix approach: [plan]"
- Implement fix
- "Fix applied! Issue resolved. Test steps: [list]"
