# API Versioning

Set up API versioning strategy and manage API versions.

## What This Command Does

Sets up API versioning by:
- Defining versioning strategy (URL, header, etc.)
- Creating version management
- Setting up version routing
- Documenting versioning approach
- Managing deprecated versions

## How It Works

1. **Understand Versioning Needs**
   - Ask about versioning strategy
   - Identify API versions
   - Understand deprecation policy
   - Determine routing approach

2. **Design Versioning Strategy**
   - Choose versioning approach
   - Set up version routing
   - Plan deprecation
   - Design migration path

3. **Generate Configuration**
   - Create versioning config
   - Set up routing
   - Configure deprecation
   - Document strategy

## What to Tell the User

- "What versioning strategy? (URL path, header, query param)"
- "What is your deprecation policy?"
- Generate API versioning configuration
- "API versioning setup complete! Strategy: [type]. Versions: [managed]. Deprecation: [configured]"
