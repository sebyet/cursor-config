# Create Release Notes

Generate release notes from commits, PRs, and changes for a release.

## What This Command Does

Creates release notes by:
- Analyzing commits and PRs
- Categorizing changes (features, fixes, breaking changes)
- Generating formatted release notes
- Including contributors and links
- Creating changelog entries

## How It Works

1. **Understand Release Scope**
   - Ask about release version
   - Identify version range
   - Understand release type
   - Determine format preferences

2. **Analyze Changes**
   - Extract commits and PRs
   - Categorize changes
   - Identify breaking changes
   - Collect contributor info

3. **Generate Release Notes**
   - Format release notes
   - Include categories
   - Add links and details
   - Create changelog entry

## What to Tell the User

- "What version is this release? (e.g., v1.2.0)"
- "What commits or PRs should we include?"
- Generate release notes
- "Release notes created! Version: [version]. Changes: [count]. Format: [markdown/plain]"
