# Monitor Errors

Set up error monitoring with Sentry or similar tools to track and alert on production errors.

## What This Command Does

Sets up error monitoring by:
- Integrating error tracking (Sentry)
- Configuring error alerts
- Setting up error reporting
- Creating error dashboards
- Configuring error filtering
- Setting up notifications

## How It Works

1. **Understand Error Monitoring Needs**
   - Ask about error tracking requirements
   - Identify error sources
   - Understand alert preferences
   - Get notification requirements

2. **Set Up Error Tracking**
   - Install error tracking SDK
   - Configure error capture
   - Set up source maps
   - Configure error context
   - Add user context
   - Set up release tracking

3. **Configure Error Alerts**
   - Set up alert rules
   - Configure alert thresholds
   - Set up notification channels
   - Configure alert filtering
   - Add alert routing

4. **Set Up Error Reporting**
   - Configure error grouping
   - Set up error dashboards
   - Add error analytics
   - Configure error filtering
   - Set up error workflows

5. **Verify Error Monitoring**
   - Test error capture
   - Verify alerts
   - Check notifications
   - Test error reporting
   - Document setup

## What to Tell the User

- "What error tracking tool do you want to use?"
- "What errors should trigger alerts?"
- Set up error monitoring
- "Error monitoring setup complete! Tool: [name]. Alerts: [configured]. Dashboard: [url]"
