# Describe Project

Looks through all your project files and notes to create a simple description of what your project does. Creates a title, a description, and a list of features - all written in simple words that anyone can understand.

1. **Read project notes**
   - Check if there's a folder with project notes
   - Read all the notes files you've written
   - Look for information about what your project does
   - Understand the history of your project
   - Figure out what your project is for

2. **Look through your project**
   - Check all the pages and screens in your project
   - Find what features exist by looking at what you've built
   - Look at the names of things to understand what they do
   - Find the main things users can do (like logging in, viewing dashboards, filling out forms, etc.)
   - Identify the main areas where people can do things

3. **Read project information**
   - Check your project settings file for the project name
   - Check if there's already a description file
   - Look for any notes about what the project does
   - Check project files that might tell you what it's for

4. **Create a simple description**
   - **Title:** Make a short, clear project title (1-10 words)
     - Based on the project name or what it mainly does
     - Easy to understand
   - **Description:** Write a simple project description (2-4 sentences)
     - Explain what the project does in everyday words
     - Focus on why it's useful for people
     - Use simple words anyone can understand
     - Mention what makes it helpful
   - **Features List:** Make a list of the main things it can do
     - Find features by looking at what's built
     - Write each feature from a user's point of view (what they can do)
     - Use simple, clear words
     - Focus on what it does, not how it's made
     - Group similar features together if it helps
     - Put the most important features first

5. **Show the description**
   - Show the title, description, and features you created
   - Make it easy to read
   - Ask if they want to save it to a description file
   - Let them change or improve the description if they want

## How to Write It

### Write for Regular People
- Write from a regular person's point of view ("You can...", "People can...")
- Focus on what people can do, not how it was built
- Use everyday words that everyone knows
- Talk about why it's helpful

### How to Describe Features
- Start with action words ("Create", "Manage", "Track", "Share")
- Be clear but keep it short
- Point out what makes it special or useful
- Don't mention technical things like what tools were used to build it

## What to Tell the User

- Explain you're looking through their project files and notes
- Tell them what you're doing: "Reading your project notes... Looking through your project files... Finding what features you have..."
- Create the description based on what you find
- Show them the title, description, and features you created
- Ask if they'd like to:
  - Save it to a description file
  - Save it somewhere else
  - Change or improve the description
- Make changes if they want something different
