# Create ADR

Create Architecture Decision Records (ADRs) to document important technical decisions.

## What This Command Does

Creates ADRs by:
- Documenting architecture decisions
- Recording decision context
- Documenting alternatives considered
- Recording consequences
- Maintaining decision history

## How It Works

1. **Understand ADR Needs**
   - Ask about decision to document
   - Identify decision context
   - Understand alternatives
   - Determine decision impact

2. **Create ADR**
   - Document decision
   - Record context
   - List alternatives
   - Document consequences

3. **Save and Organize**
   - Save ADR file
   - Organize in ADR directory
   - Update ADR index
   - Document decision

## What to Tell the User

- "What architecture decision should we document?"
- "What alternatives were considered?"
- Create ADR and save
- "ADR created! Decision: [title]. File: [path]. Index: [updated]"
