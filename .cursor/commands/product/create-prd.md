# Create PRD

Generate product requirement documents (PRDs) that clearly define features, requirements, and success criteria.

## What This Command Does

Creates comprehensive PRDs that include:
- Problem statement and user needs
- Goals and success metrics
- Feature requirements and specifications
- User stories and acceptance criteria
- Technical considerations
- Timeline and dependencies

## How It Works

1. **Gather Requirements**
   - Ask about the feature or product
   - Understand the problem being solved
   - Identify target users
   - Clarify business goals

2. **Structure the PRD**
   - Problem statement
   - Goals and success metrics
   - User personas and use cases
   - Feature requirements
   - Technical specifications
   - Design considerations
   - Timeline and milestones
   - Risks and dependencies

3. **Generate the Document**
   - Create well-structured PRD
   - Include all necessary sections
   - Format for clarity and review

## What to Tell the User

- "What feature or product do you want to create a PRD for?"
- "What problem does this solve? Who is it for?"
- "What are the key goals and success metrics?"
- Generate the PRD document
- "PRD created! Review and refine as needed."
