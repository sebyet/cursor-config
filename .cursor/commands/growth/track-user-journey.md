# Track User Journey

Track user journey analytics to understand user behavior and optimize experiences.

## What This Command Does

Tracks user journeys by:
- Defining journey stages
- Setting up event tracking
- Creating analytics implementation
- Configuring funnel analysis
- Generating tracking plan

## How It Works

1. **Understand User Journey**
   - Ask about user journey to track
   - Identify key stages
   - Understand user actions
   - Get analytics platform

2. **Design Tracking**
   - Map journey stages
   - Define events to track
   - Create tracking plan
   - Configure analytics

3. **Generate Implementation**
   - Create tracking configuration
   - Document setup steps
   - Provide implementation guide
   - Add validation checks

## What to Tell the User

- "What user journey do you want to track?"
- "What are the key stages and events?"
- Generate tracking implementation
- "User journey tracking setup complete! Events: [list]. Implementation: [details]"
