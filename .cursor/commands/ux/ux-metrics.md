# UX Metrics

Define and track UX metrics and KPIs to measure user experience success and improvement.

## What This Command Does

Manages UX metrics by:
- Defining UX metrics and KPIs
- Setting up UX measurement framework
- Tracking UX metrics over time
- Analyzing UX metric trends
- Reporting on UX performance

## How It Works

1. **Define UX Metrics**
   - Ask about UX goals and objectives
   - Identify key UX metrics (task success rate, time on task, error rate, satisfaction, etc.)
   - Define measurement methods
   - Set target values and benchmarks
   - Determine tracking frequency

2. **Set Up Measurement**
   - Plan data collection methods
   - Set up tracking and analytics
   - Configure measurement tools
   - Create baseline measurements
   - Document measurement process

3. **Track and Monitor**
   - Collect UX metric data
   - Track metrics over time
   - Monitor trends and changes
   - Identify metric improvements or declines
   - Document metric performance

4. **Analyze and Report**
   - Analyze UX metric trends
   - Compare against targets and benchmarks
   - Identify areas for improvement
   - Generate UX metrics report
   - Recommend actions based on metrics

## What to Tell the User

- "What UX goals do you want to measure?"
- "What UX metrics are important? (task success, time on task, satisfaction, etc.)"
- "What's your measurement method?"
- Set up and track UX metrics
- "UX metrics setup complete! Metrics: [list]. Baseline: [established]. Tracking: [configured]"
