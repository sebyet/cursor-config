# Service Blueprint

Create service blueprints to map service experiences end-to-end including frontstage and backstage processes.

## What This Command Does

Creates service blueprints by:
- Mapping service experiences end-to-end
- Identifying service touchpoints
- Documenting frontstage and backstage processes
- Finding service improvement opportunities
- Aligning teams on service delivery

## How It Works

1. **Define Service Scope**
   - Ask about service to blueprint
   - Identify user journey and scenario
   - Understand service context
   - Determine blueprint scope
   - Get context on existing service

2. **Map Service Experience**
   - Map user actions and touchpoints
   - Document frontstage processes (visible to user)
   - Map backstage processes (behind the scenes)
   - Identify support processes
   - Document physical evidence

3. **Identify Service Elements**
   - Map customer journey line
   - Document interaction line
   - Map visibility line
   - Identify internal interaction line
   - Document support processes

4. **Analyze and Improve**
   - Identify pain points and friction
   - Find service gaps and failures
   - Identify improvement opportunities
   - Suggest service optimizations
   - Create service blueprint document

## What to Tell the User

- "What service do you want to blueprint?"
- "What's the user journey or scenario?"
- "What's the service context?"
- Create service blueprint
- "Service blueprint created! Touchpoints: [count]. Pain points: [list]. Improvements: [list]"
