# User Story Mapping

Create user story maps to organize features by user stories and plan releases.

## What This Command Does

Creates user story maps by:
- Organizing features by user stories
- Mapping user activities and tasks
- Prioritizing features and releases
- Planning MVP and feature roadmaps
- Aligning team on user value

## How It Works

1. **Understand Product Goals**
   - Ask about product or feature set
   - Identify user personas and goals
   - Understand business objectives
   - Get context on existing features
   - Determine mapping scope

2. **Map User Activities**
   - Identify high-level user activities
   - Break down activities into tasks
   - Organize tasks by user story
   - Map user journey through activities
   - Document user goals at each level

3. **Prioritize Stories**
   - Identify MVP stories (walking skeleton)
   - Prioritize by user value
   - Group stories into releases
   - Plan release roadmap
   - Align on priorities

4. **Generate Story Map**
   - Create visual story map
   - Document activities and tasks
   - Show prioritization and releases
   - Include user goals and value
   - Make it actionable for planning

## What to Tell the User

- "What product or feature set do you want to map?"
- "Who are the users? What are their goals?"
- "What's the scope of the story map?"
- Create user story map
- "User story map created! Activities: [count]. MVP stories: [list]. Release plan: [summary]"
