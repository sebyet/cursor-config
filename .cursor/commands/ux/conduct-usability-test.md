# Conduct Usability Test

Conduct usability testing sessions to validate designs and identify usability issues with real users.

## What This Command Does

Conducts usability testing by:
- Planning usability test sessions
- Creating test scenarios and tasks
- Facilitating usability tests (moderated/unmoderated)
- Analyzing usability test results
- Identifying usability issues and prioritizing fixes
- Generating usability test reports

## How It Works

1. **Plan Usability Test**
   - Ask what to test (feature, page, flow)
   - Identify test objectives and research questions
   - Determine test type (moderated, unmoderated, remote, in-person)
   - Define participant criteria and recruitment needs
   - Plan test scenarios and tasks

2. **Create Test Materials**
   - Design test scenarios and tasks
   - Create test scripts and guides
   - Prepare test environment and tools
   - Set up recording and observation methods
   - Create consent forms and documentation

3. **Conduct Usability Test**
   - Facilitate test sessions
   - Observe user behavior and interactions
   - Note usability issues and pain points
   - Record user feedback and comments
   - Document findings in real-time

4. **Analyze Results**
   - Review test recordings and notes
   - Identify usability issues and patterns
   - Prioritize issues by severity and frequency
   - Extract insights and recommendations
   - Generate usability test report

## What to Tell the User

- "What do you want to test for usability?"
- "What are your test objectives?"
- "What type of usability test? (moderated, unmoderated, remote, in-person)"
- Plan and conduct usability test
- "Usability test complete! Issues found: [count]. Critical issues: [list]. Recommendations: [list]"
