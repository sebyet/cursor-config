# Debug Issue

Finds and fixes bugs in your code. Investigates what's wrong, identifies the root cause, and fixes it so your website works correctly.

1. **Understand the problem**
   - Ask what's not working or what error they're seeing
   - Reproduce the issue if possible
   - Check error messages and logs
   - Identify where the problem occurs
   - Tell the user what you found

2. **Investigate the bug**
   - Check the relevant code
   - Look for logical errors
   - Check data flow and state management
   - Check API calls and responses
   - Check component rendering
   - Find the root cause

3. **Fix the bug**
   - Fix the root cause
   - Test the fix
   - Make sure nothing else broke
   - Verify the issue is resolved

4. **Verify the fix**
   - Test that the bug is gone
   - Make sure related features still work
   - Confirm everything is working correctly

## What to Tell the User

- Ask what's not working: "What's the problem? What error are you seeing?"
- Explain you're investigating the issue
- Show what you found: "I found the issue in [file/component]..."
- Explain the root cause in simple terms
- Fix it step by step
- Verify the fix works
- Tell them when the bug is fixed!
