# Integration Test

Create integration tests for APIs, databases, and service integrations.

## What This Command Does

Creates integration tests for:
- API endpoint integration
- Database integration
- Third-party service integration
- Service-to-service communication
- Data flow validation
- Error handling scenarios

## How It Works

1. **Understand Integration Needs**
   - Ask about integration to test
   - Identify integration points
   - Understand data flow
   - Get context on services

2. **Create Test Plan**
   - Define test scenarios
   - Identify test cases
   - Plan test data
   - Set up test environment
   - Document test plan

3. **Set Up Test Environment**
   - Configure test database
   - Set up test services
   - Configure test data
   - Set up mocking if needed
   - Configure test utilities

4. **Write Integration Tests**
   - Write API integration tests
   - Create database integration tests
   - Test service integrations
   - Add error scenario tests
   - Test data validation

5. **Run and Verify Tests**
   - Run integration tests
   - Verify test coverage
   - Check test reliability
   - Fix any test issues
   - Document test results

## What to Tell the User

- "What integration do you want to test?"
- "What services are involved?"
- Create integration tests
- "Integration tests created! Test scenarios: [count]. Coverage: [details]. Run tests with: [command]"
