# API Test

Create API tests for endpoints, contracts, performance, and security.

## What This Command Does

Creates API tests for:
- API endpoint testing
- API contract testing
- API performance testing
- API security testing
- API error handling
- API data validation

## How It Works

1. **Understand API Testing Needs**
   - Ask about APIs to test
   - Identify test objectives
   - Understand API structure
   - Get context on requirements

2. **Create Test Plan**
   - Define test scenarios
   - Identify test cases
   - Plan test data
   - Set up test environment
   - Document test plan

3. **Set Up API Testing**
   - Configure API client
   - Set up test utilities
   - Configure test data
   - Set up authentication
   - Configure test environment

4. **Write API Tests**
   - Write endpoint tests
   - Create contract tests
   - Add performance tests
   - Test security scenarios
   - Test error handling
   - Validate responses

5. **Run and Verify Tests**
   - Run API tests
   - Verify test coverage
   - Check test reliability
   - Fix any test issues
   - Document test results

## What to Tell the User

- "What APIs do you want to test?"
- "What test objectives do you have?"
- Create API tests
- "API tests created! Endpoints: [count]. Test scenarios: [count]. Run tests with: [command]"
