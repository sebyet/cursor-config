# Security Test

Create security tests including vulnerability testing, penetration testing support, and security test automation.

## What This Command Does

Creates security tests for:
- Security test scenarios
- Vulnerability testing
- Penetration testing support
- Security test automation
- Authentication testing
- Authorization testing

## How It Works

1. **Understand Security Testing Needs**
   - Ask about security concerns
   - Identify test objectives
   - Understand security requirements
   - Get context on needs

2. **Create Test Plan**
   - Define security scenarios
   - Identify test cases
   - Plan security testing
   - Set up test environment
   - Document test plan

3. **Set Up Security Testing**
   - Configure security tools
   - Set up test scenarios
   - Configure test data
   - Set up security monitoring
   - Configure test environment

4. **Write Security Tests**
   - Write vulnerability tests
   - Create authentication tests
   - Test authorization
   - Add penetration test scenarios
   - Test security controls

5. **Run and Analyze Tests**
   - Run security tests
   - Analyze security findings
   - Identify vulnerabilities
   - Prioritize security issues
   - Generate security report

## What to Tell the User

- "What security testing do you need?"
- "What security concerns do you have?"
- Create security tests
- "Security tests created! Scenarios: [count]. Vulnerabilities found: [count]. Report: [generated]"
