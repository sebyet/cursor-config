# Visual Regression Test

Perform visual regression testing to detect unintended UI changes.

## What This Command Does

Performs visual regression testing by:
- Capturing UI screenshots
- Comparing against baseline images
- Detecting visual differences
- Identifying layout issues
- Generating visual diff reports

## How It Works

1. **Understand Visual Testing Needs**
   - Ask about pages/components to test
   - Identify browsers/devices
   - Understand visual requirements
   - Determine test scope

2. **Design Visual Tests**
   - Create test scenarios
   - Set up screenshot capture
   - Define comparison rules
   - Plan baseline management

3. **Execute and Analyze**
   - Run visual tests
   - Compare screenshots
   - Detect differences
   - Generate reports

## What to Tell the User

- "What pages or components should we test visually?"
- "What browsers and devices should we test?"
- Execute visual regression tests and analyze results
- "Visual regression test complete! Tests: [count]. Differences: [list]. Reports: [generated]"
