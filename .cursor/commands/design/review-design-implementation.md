# Review Design Implementation

Check if code implementation matches design specifications and design system guidelines.

## What This Command Does

Reviews design implementation by:
- Comparing code to design specs
- Checking design system compliance
- Verifying spacing, colors, typography
- Identifying visual discrepancies
- Ensuring responsive behavior matches design

## How It Works

1. **Gather Design Context**
   - Ask about design files or specs
   - Identify components to review
   - Understand design system
   - Get access to design files

2. **Review Implementation**
   - Compare code to design
   - Check design tokens (colors, spacing, typography)
   - Verify component structure
   - Test responsive behavior
   - Check accessibility

3. **Generate Review Report**
   - List discrepancies found
   - Prioritize issues
   - Suggest fixes
   - Provide specific recommendations

## What to Tell the User

- "What component or page do you want to review?"
- "Do you have design files or specs to compare against?"
- Review implementation and compare to design
- "Design review complete! Issues found: [list]. Fixes needed: [list]"
