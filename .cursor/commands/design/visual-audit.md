# Visual Audit

Perform comprehensive visual design quality review focusing on aesthetic quality, visual hierarchy, and overall design polish.

## What This Command Does

Performs visual audits by:
- Reviewing visual hierarchy and information architecture
- Assessing visual balance and composition
- Evaluating color harmony and aesthetic contrast
- Reviewing typography choices and readability
- Assessing spacing, alignment, and whitespace
- Identifying visual design improvements

## How It Works

1. **Define Audit Scope**
   - Ask what to audit (pages, components, features)
   - Understand design goals and brand identity
   - Identify focus areas (hierarchy, balance, typography, etc.)
   - Get context on target audience

2. **Review Visual Design**
   - Analyze visual hierarchy and information flow
   - Assess composition and visual balance
   - Review color usage and harmony
   - Evaluate typography choices and hierarchy
   - Check spacing, alignment, and whitespace usage
   - Assess overall aesthetic quality and polish

3. **Identify Issues and Opportunities**
   - Find visual hierarchy problems
   - Identify composition issues
   - Note color harmony concerns
   - Flag typography readability issues
   - Highlight spacing and alignment problems
   - Suggest improvements

4. **Generate Audit Report**
   - Create comprehensive audit document
   - List issues by priority
   - Provide specific recommendations
   - Include visual examples where possible
   - Suggest actionable improvements

## What to Tell the User

- "What do you want to audit visually?"
- "What are your design goals and brand identity?"
- "What areas should we focus on?"
- Review visual design quality
- "Visual audit complete! Issues found: [list]. Recommendations: [list]"
