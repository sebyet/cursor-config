# Competitive Design Analysis

Analyze competitor designs to extract insights, identify patterns, and inform design decisions.

## What This Command Does

Performs competitive design analysis by:
- Analyzing competitor UI/UX patterns
- Identifying design trends and best practices
- Comparing design approaches across competitors
- Extracting actionable design insights
- Documenting findings and recommendations

## How It Works

1. **Define Analysis Scope**
   - Ask which competitors to analyze
   - Identify specific features or pages to compare
   - Understand analysis goals (what to learn)
   - Determine focus areas (UI patterns, UX flows, visual design)

2. **Analyze Competitor Designs**
   - Review competitor websites/apps
   - Document UI patterns and components
   - Analyze user flows and interactions
   - Identify visual design approaches
   - Note strengths and weaknesses
   - Compare across multiple competitors

3. **Extract Insights**
   - Identify common patterns and trends
   - Highlight innovative approaches
   - Note best practices
   - Find opportunities for differentiation
   - Document design decisions and rationale

4. **Generate Analysis Report**
   - Create comprehensive analysis document
   - Include visual comparisons
   - Document findings and insights
   - Provide actionable recommendations
   - Suggest design opportunities

## What to Tell the User

- "Which competitors do you want to analyze?"
- "What features or pages should we focus on?"
- "What are you trying to learn from this analysis?"
- Analyze competitor designs and extract insights
- "Competitive analysis complete! Key findings: [list]. Recommendations: [list]"
