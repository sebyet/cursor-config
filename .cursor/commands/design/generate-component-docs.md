# Generate Component Docs

Create component documentation including usage, props, examples, and guidelines.

## What This Command Does

Generates component documentation that includes:
- Component description and purpose
- Props and API documentation
- Usage examples
- Design guidelines
- Accessibility notes
- Best practices

## How It Works

1. **Identify Components**
   - Ask which components to document
   - Understand component purpose
   - Get component code

2. **Analyze Component**
   - Review component code
   - Extract props and API
   - Understand usage patterns
   - Identify design considerations

3. **Generate Documentation**
   - Create comprehensive docs
   - Include examples
   - Add usage guidelines
   - Document best practices

## What to Tell the User

- "What components do you want to document?"
- Analyze components and generate docs
- "Component documentation generated! Docs created for: [list]"
