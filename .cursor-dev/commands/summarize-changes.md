# Summarize Changes

## Overview
Provide a clear summary of what was implemented in the feature branch, how it compares to the main branch, and any key differences, improvements, or updates. This helps the team understand the featureâ€™s scope, impact, and readiness for merge. R

## Steps
1. **Compare Feature Branch to Main**
   - Highlight added, modified, or removed functionality
   - Note differences in behavior, APIs, or configuration
   - Identify any potential conflicts or deviations from main

2. **Summarize Implementation**
   - Provide a high-level overview of what was done in the branch
   - Include references to tickets, tasks, or plan work items
   - Mention important fixes, features, or improvements

3. **Highlight Key Points**
   - Note any limitations or known issues
   - Indicate if README updates are required
   - Reference changelog updates

## Checklist
- [ ] Key differences between feature branch and main identified
- [ ] High-level overview of changes documented
- [ ] References to tickets, plan work, or tasks included
- [ ] Any required README or changelog updates noted
